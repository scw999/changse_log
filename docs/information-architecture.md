# 창세록 Information Architecture

## Product Frame

창세록은 메모 앱이 아니라 "구조화된 개인 기록 아카이브"다. 입력의 시작점은 웹앱이 아니라 텔레그램과 창세봇이며, 웹앱의 역할은 저장, 분류, 탐색, 회고, 재발견에 있다.

## Core Navigation

1. 대시보드 `/`
2. 최근 기록 `/recent`
3. 생각 `/thoughts`
4. 단어 `/words`
5. 콘텐츠 `/content`
6. 장소 `/places`
7. 활동 `/activities`
8. 리뷰 `/review`
9. 관리자 편집기 `/admin`
10. 기록 상세 `/records/[id]`

## Primary Jobs To Be Done

- 방금 저장된 기록을 빠르게 다시 찾는다.
- 카테고리, 태그, 날짜, 평점, 지역 기준으로 기록을 좁힌다.
- 장소, 콘텐츠, 활동의 평점을 비교하면서 다시 볼 가치가 있는 대상을 발견한다.
- 생각과 단어를 시간이 지난 뒤 다시 읽으면서 패턴을 찾는다.
- 관리 화면에서 구조화된 형태로 기록을 수정하거나 신규 기록을 등록한다.

## Page Responsibilities

### Dashboard

- 최근 기록 요약
- 카테고리별 현황
- 평점 높은 장소/콘텐츠
- 다시 보고 싶은 기록
- 최근 생각, 최근 장소, 최근 활동 하이라이트

### Recent / Inbox

- 전체 기록 타임라인
- 최신순, 오래된순, 평점순, 중요도순 정렬
- 검색, 태그, 세부 유형, 지역, 평점, 중요도 필터

### Category Pages

- `thoughts`: 생각, 아이디어, 회고, 메모
- `words`: 단어, 표현, 조어, 남겨둘 문장
- `content`: 영화, 책, 드라마, 영상
- `places`: 식당, 카페, 여행지, 데이트 장소, 방문 장소
- `activities`: 운동, 등산, 러닝, 산책, 트래킹 기록

각 카테고리 페이지는 공통적으로 목록 탐색 중심이며, 카테고리별 메타데이터를 카드와 상세 페이지에 노출한다.

### Review

- 즐겨찾는 기록
- 고평점 콘텐츠
- 재방문 후보 장소
- 다시 읽을 생각/메모
- 월별 타임라인 기반 회고

### Admin

- 신규 기록 작성
- 기존 기록 수정/삭제
- 카테고리별 구조 필드 입력
- 태그와 중요도 관리
- 향후 창세봇 승인 흐름을 연결할 편집 진입점

## Record Detail Model

모든 상세 페이지는 공통 메타 정보와 카테고리별 상세 정보로 구성한다.

- 공통: 제목, 요약, 본문, 태그, 생성일, 사건일, 중요도, 저장 출처
- 카테고리 상세:
  - 생각: 한 줄 생각, 확장 노트, 액션 필요 여부, 다시 볼 가치
  - 단어: 뜻, 예문, 저장 이유
  - 콘텐츠: 원제, 평점, 한 줄 리뷰, 기억 포인트, 약점, 인상 문장, 재감상 의도
  - 장소: 지역, 주소, 방문일, 평점, 동행, 분위기, 가격 메모, 재방문 의도
  - 활동: 거리, 시간, 난이도, 만족도, 컨디션 메모

## Data Flow

1. 사용자 -> 텔레그램 -> 창세봇
2. 창세봇 -> 구조화 텍스트 제안
3. 사용자 승인
4. 창세록 저장
5. 창세록에서 탐색, 필터링, 회고, 수정

Phase 1에서는 4번 저장 계층을 로컬 시드 데이터와 브라우저 저장소로 구현하고, Phase 2에서 Supabase 저장소로 교체한다.
